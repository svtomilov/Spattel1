# Spattel — сайт

Одностраничный лендинг «Капитальный ремонт вторички под ключ с личным руководителем проекта».
Чистый HTML/CSS/JS, без фреймворков и сборки. Анимации на скролле — GSAP + ScrollTrigger (CDN).

## Структура

```
site/
├── index.html          # главная страница
├── admin.html           # редактор текстов/цен (не в навигации, см. раздел ниже)
├── content.json         # редактируемые тексты/цены — подставляются на страницу из js/main.js
├── 404.html             # страница «не найдено»
├── robots.txt
├── sitemap.xml
├── css/style.css
├── js/main.js           # подстановка content.json, скролл-скраб hero-видео, reveal-анимации, форма лида, мобильная sticky-CTA
└── assets/
    ├── favicon.svg
    ├── hero-transform.mp4      # видео «убитая вторичка → лофт», скраб по скроллу в #hero3d
    └── placeholder-photo.svg   # временная заглушка вместо реальных фото
```

## Редактирование текстов и цен без кода — /admin.html

Основные тексты (заголовок и подзаголовок hero, подписи кнопок, цифры доверия,
названия/описания/цены трёх тарифов, вопросы-ответы FAQ) лежат в `content.json`.
`js/main.js` подставляет их на страницу при загрузке; строки в `index.html` —
запасной вариант на случай недоступности JSON.

Как отредактировать:

1. Понадобится GitHub personal access token с правом записи в этот репозиторий:
   github.com → Settings → Developer settings → Personal access tokens →
   **classic**-токен со scope `repo` (или fine-grained с Contents: Read and write
   для `svtomilov/Spattel1`).
2. Откройте `/admin.html` на опубликованном домене (страница не проиндексирована
   и не связана ссылками с сайтом — только прямой адрес).
3. Вставьте токен, нажмите «Загрузить контент», отредактируйте поля и нажмите
   «Сохранить и опубликовать».

Сохранение делает коммит `content.json` прямо в ветку `main` через GitHub API —
Vercel подхватывает его и публикует сайт заново примерно за минуту. Токен никуда
не записывается, кроме sessionStorage текущей вкладки: после закрытия браузера
его нужно вводить заново. Не коммитьте токен в репозиторий и не встраивайте его
в код.

## Деплой

### Vercel
1. `npm i -g vercel` (если не установлен)
2. В папке `site/` выполнить `vercel` и следовать подсказкам — root directory уже статический, build command не нужен
3. Для продакшна: `vercel --prod`

### Netlify
1. Перетащить папку `site/` в Netlify Drop (app.netlify.com/drop), либо
2. Подключить репозиторий, указать **Base directory:** `site`, **Publish directory:** `site` (build command — пусто, это статика)

### Любой статический хостинг
Просто скопировать содержимое `site/` на сервер — зависимостей и шага сборки нет.
